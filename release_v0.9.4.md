## Что нового

`v0.9.4` — патч-релиз безопасности и надёжности OneBase. Интерфейс теперь подстраивается под права пользователя, чувствительные поля можно маскировать по ролям с контролируемым раскрытием, а прикладные правила валидации — публиковать по версиям и бизнес-датам. Для Windows улучшен запуск Предприятия, DSL получил ручную атомарную нумерацию. Исправлены пометка на удаление и физическое удаление объектов в транзакциях PostgreSQL.

## Изменения

- **Маскирование персональных данных** — роли поддерживают политики `mask_tail`, `mask_city`, `mask_all`, `hide` и `full` для полей справочников, документов и регистров. Маска применяется на сервере в формах, списках, REST API, выборе ссылок, печати и экспорте.
- **Контролируемое раскрытие** — полное значение чувствительного поля выдаётся только с правом `disclose` и обязательным основанием; событие записывается в журнал без сохранения самого значения.
- **Версионируемые правила валидации** — пример `callcenter` показывает публикацию правил с бизнес-даты, блокирующие проверки, предупреждения и неизменяемую историю версий с откатом через новую публикацию.
- **Интерфейс по ролям** — панель разделов, рабочие столы, виджеты и быстрые действия скрывают недоступные пользователю объекты; для подсистем появился whitelist `roles`.
- **Нативное окно Предприятия** — в Windows GUI база открывается в отдельном WebView2-окне без адресной строки и с общей сессией лаунчера.
- **Ручная нумерация из DSL** — `Нумераторы.СледующийНомер("Сущность"[, Дата])` использует тот же атомарный счётчик, что UI и REST, и доступен в обработках и регламентных заданиях.
- **Атомарные DSL-хуки** — записи объектов, движения, нумераторы и очередь обмена из хуков записи и проведения участвуют в общей транзакции и вместе откатываются при ошибке.
- **Восстановление состояния обмена** — universal backup сохраняет очереди, участников и водяные знаки; при импорте можно явно выбрать восстановление того же узла или безопасный изолированный клон без прежней идентичности.

## Исправления

- Пометка на удаление и физическое удаление документов и обычных справочников больше не переводят PostgreSQL-транзакцию в аварийное состояние из-за отсутствующих служебных колонок.
- DSL и UI/REST теперь одинаково выполняют отмену проведения и сохраняют изменения шапки из `ОбработкаПроведения`; ошибка `ОбработкаУдаленияПроведения` откатывает движения и признак проведения.
- При создании и немедленном проведении документа его шапка записывается до хука, поэтому обратные ссылки не нарушают внешний ключ.
- Введённый вручную номер документа больше не перезаписывается автономером; документы без строкового реквизита получают читаемое представление вместо UUID в выборе ссылок.
- Закрыты обходы маскирования через догрузку выбранных ссылок, печатные формы, отчёты, AI и кэш виджетов; раскрытие работает по принципу fail-closed при ошибке аудита.
- Обмен между базами сохраняет версии объектов и строже проверяет топологию и участников узла-хаба.
- Обновлена зависимость `golang.org/x/text`, закрывающая актуальное предупреждение безопасности Go.

## Важные изменения поведения

> [!IMPORTANT]
> Раскрытие замаскированного значения без успешной записи в журнал регистрации теперь запрещено: API возвращает ошибку и не отдаёт исходные данные.

- Опубликованные версии правил валидации нельзя изменить или удалить отменой проведения; откат оформляется новой публикацией.
- Пользователь видит только те подсистемы и виджеты, к объектам которых у него есть права. Администратор по-прежнему видит всё.
- Режим восстановления «Клон» очищает идентичность и очереди обмена; режим «Тот же узел» восстанавливает их из архива. Секретные токены не переносятся ни в одном режиме.

## Установка

### Windows
1. Скачайте `onebase-windows-amd64.zip` и файл `onebase-windows-amd64.zip.sha256`
2. Проверьте контрольную сумму и распакуйте архив, например в `C:\onebase\`
3. Запустите `onebase-gui.exe`

### Linux
```bash
tar xzf onebase-linux-amd64.tar.gz
./onebase-linux-amd64/onebase start
```

### macOS
```bash
tar xzf onebase-darwin-amd64.tar.gz
./onebase-darwin-amd64/onebase start
```

### Offline-обновление
```bash
onebase update --from D:\flash\onebase-windows-amd64.zip --sha256 <ожидаемый-hex> --id <база>
```
