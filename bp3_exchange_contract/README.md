# Контракт обмена OneBase → БП3 (пакет для параллельной разработки)

Это **спецификация-цель** для разработки выгрузки **OneBase → 1С:Бухгалтерия (БП3)**.
Источник (OneBase, Go + SQLite) читает свою базу и **выгружает JSON в этом формате**;
готовый 1С-загрузчик БП3 этот JSON принимает и создаёт документы. 1С/COM на стороне
OneBase **не нужны** — шов интеграции чисто файловый (JSON).

> Эту папку можно разрабатывать в отдельной сессии параллельно с 1С-обменами:
> пересечение — только финальная загрузка в БП3 (1С), её сериализуют с другими 1С-прогонами.
> Сам экспортёр OneBase→JSON и валидация формата — полностью автономны.

## Файлы пакета

| Файл | Что |
|---|---|
| `schema_dump.json` | JSON Schema целевого формата (`queryDump`). Источник истины. |
| `транспорты.json` | Какие типы документов поддерживает JSON-транспорт загрузчика (растёт). |
| `пример_дампа_минимальный.json` | Рабочий пример дампа (структура `queries`/секций/строк). |
| `пример_блока_настройки.json` | Пример блока `настройки` (боевой, из АА-выгрузки). |
| `validate_dump.py` | Эталонный валидатор: схема + совместимость. `python validate_dump.py out.json` |

## Формат (кратко)

Верхний уровень:
```json
{
  "version": "1.0",
  "format": "queryDump",
  "source": "OneBase-PuT",
  "exported": "2026-06-19T10:00:00",
  "period": { "from": "2026-01-01", "to": "2026-03-31" },
  "queries": { "<Секция>": { "columns": ["..."], "rows": [["..."], ...] } },
  "настройки": { ... }
}
```

- **`queries`** — словарь секций. Ключ = имя раздела (справочника/документа). Значение —
  табличный результат: `columns` (имена колонок) + `rows` (массив строк, каждая — массив
  значений по числу колонок). Значения примитивны (строка/число/булево/null; даты — ISO-строкой).
- **Известные секции** валидируются СТРОГО по составу колонок (`Контрагенты`, `Номенклатура`,
  `Склады`, `СтатьиДДС`, документные `РеализацияТоваровУслуг` и др. — см. `$defs` в схеме).
  Неизвестные секции — generic (`columns`/`rows`), без контракта колонок.

### ⚠️ Главное правило: денормализация
Загрузчик БП3 резолвит ссылки **офлайн из дампа** (живого источника при загрузке нет).
Поэтому на каждую ссылочную колонку дамп обязан нести:
- парную колонку **`<Колонка>УИН`** — строковый GUID объекта-источника (ключ резолвинга);
- + вычисляемые/навигационные поля, которые иначе пришлось бы добирать у источника
  (наименование, код, ИНН, ставка НДС именем, валюта и т.п.).

Состав обязательных колонок известных секций — в `schema_dump.json` (`contains` внутри `$defs`).
Смотри `пример_дампа_минимальный.json` как живой образец (УИН-пары видны там).

### Блок `настройки` (опционально, но желателен)
Несёт выбор оператора + происхождение (`пример_блока_настройки.json`):
```json
"настройки": {
  "организации": ["ИНН1"],
  "объекты": ["РеализацияТоваровУслуг"],
  "источник": { "конфигурация": "OneBase-PuT", "версияКонфигурации": "...", "имяБазы": "PUTDemo" },
  "экспортёр": { "версия": "...", "собран": "2026-06-19" },
  "транспорт": "JSON"
}
```
- `транспорт` ∈ {"COM","JSON"} — для OneBase всегда **"JSON"**.
- `источник.конфигурация` — стабильное имя источника (используется загрузчиком для сверки).

## Как валидировать вывод OneBase
```bash
python validate_dump.py путь/к/выгрузке.json OneBase-PuT
# [schema] OK
# [совместимость] OK
```
`pip install jsonschema` — для полной схемной проверки (без него идёт только структурная).
Схема языконезависима: можно валидировать и Go-библиотекой jsonschema, файл `schema_dump.json`.

## Рекомендованный порядок (минимальный срез)
1. Выгрузить **справочники**: `Контрагенты`, `Номенклатура`, `Склады`, `СтатьиДДС` (с УИН/денормализацией).
2. Один документный тип, например `РеализацияТоваровУслуг` (колонки — см. `$defs/разделРТУ` в схеме).
3. Прогнать `validate_dump.py` → зелёно.
4. Передать дамп для загрузки в БП3 (1С-загрузчик) — **этот шаг координируется с 1С-прогонами** (общие базы/лицензии).

## Расширение контракта
Если у PuT есть документ без секции в схеме — добавляется новый `$def` раздела в основном
репозитории обмена (по согласованию), здесь обновляется `schema_dump.json`. Неизвестные секции
загрузчик пока не грузит (нужна per-документная ветка в загрузчике).
