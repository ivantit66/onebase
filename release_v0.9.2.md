## Что нового

`v0.9.2` — патч-релиз для распределённых и рабочих инсталляций OneBase. Появился полноценный обмен данными между базами, усилены резервное копирование, восстановление и обновление платформы, исправлена установка Windows-сервиса и упрощена диагностика в пользовательском режиме и конфигураторе.

## Изменения

- **Планы обмена между базами** — файловые пакеты `.obx` и онлайн-синхронизация по HTTP, монитор очереди, подтверждения доставки и разрешение конфликтов по времени, приоритету узла или DSL-хуку.
- **Фаза 2 обмена** — константы и непериодические регистры сведений в составе, топология «звезда» с хабом и перепроведение документов на онлайн-приёмнике через `repost: true`.
- **Надёжность обмена** — строгая адресация и проверка пакетов, лимиты, tombstone для физически удалённых объектов, повтор перепроведения после временной ошибки и детерминированное разрешение равных временных меток.
- **Бэкапы и восстановление** — проверяемые атомарные операции для SQLite и universal backup, защита ZIP-импорта от обхода путей, симлинков и чрезмерной распаковки.
- **Windows-сервис** — корректные пути с пробелами, безопасное экранирование аргументов, проверка mapped drives и подсказки по UNC-путям.
- **Константы** — серверная проверка `required`, значений перечислений и ссылок; перечисления отображаются как выпадающий список.
- **Стабильные вкладки** — одна SSE-сессия и один опрос сообщений на оболочку независимо от числа открытых внутренних вкладок.
- **Диагностика интерфейса** — окна «О программе» показывают расположения базы данных и конфигурации; PostgreSQL-пароли маскируются.
- **Меню «Система»** — пункты сгруппированы, длинное меню прокручивается, РМК доступен через «Все функции → Платформенные возможности».
- **ИИ в конфигураторе** — кнопки помощника и генерации каркаса появляются после настройки; встроенная статика больше не застревает в кэше WebView.
- **CI и релизы** — добавлены PostgreSQL integration-проверки, race/coverage-контроль и SHA256-файлы для всех релизных архивов.

## Исправления

- Усилены итоги регистров, UTC-периоды, конкурентные записи и свёртка документов с регистрацией удаления в обмене.
- Hot reload публикует новый реестр атомарно и корректно освобождает ресурсы watcher.
- Исправлены проверки доступа к строкам, вложения REST API v2, SSE-доставка и сценарии self-update.
- Устранены частичные записи констант и регистров сведений при ошибке регистрации обмена.

## Важные изменения поведения

> [!IMPORTANT]
> `onebase update` теперь требует обязательный `--sha256`. SQLite-восстановление требует остановить сервис и явно передать `--force`.

- Для плана обмена с тремя и более узлами обязателен ровно один узел с `role: hub`; плоская многоузловая топология отклоняется как небезопасная.
- Настройки уровня процесса из `app.yaml` применяются после перезапуска, а не через hot reload.

## Установка

### Windows
1. Скачайте `onebase-windows-amd64.zip` и файл `onebase-windows-amd64.zip.sha256`
2. Проверьте контрольную сумму и распакуйте архив, например в `C:\onebase\`
3. Запустите `onebase-gui.exe`

### Linux
```bash
tar xzf onebase-linux-amd64.tar.gz
./onebase-linux-amd64/onebase start
```

### macOS
```bash
tar xzf onebase-darwin-amd64.tar.gz
./onebase-darwin-amd64/onebase start
```

### Offline-обновление
```bash
onebase update --from D:\flash\onebase-windows-amd64.zip --sha256 <ожидаемый-hex> --id <база>
```
