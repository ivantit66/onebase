# План: расширения языка DSL — что есть, что нужно, что не делаем

**Статус:** ✅ Priority 1 по языку фактически реализован (сверка 2026-06-25):
`ИначеЕсли`, тернарный `?(...)`, `+=`, `-=`, `*=`, `/=`, расширенные даты/строки,
`Пустая`/`ЗначениеЗаполнено`, `Формат`, `ТипЗнч`/`Тип`. Из runtime-волны
реализованы общие модули, `ТаблицаЗначений`, `Документы.X.Создать()` и
write-hooks; главным оставшимся пунктом остаётся автодоступ к полям ссылок
`X.Y`.

## Цель

Систематический roadmap по DSL OneBase. План **отвечает на вопрос** «что должно быть в языке, чтобы pet-конфигурация писалась естественно, без обходных манёвров». Не дублирует [[34-dsl-real-world]] — тот про конкретные 6 фич для familyBg, этот — про общую языковую модель и приоритизацию.

**Якорь:** [[project_pet_focus]] — OneBase pet, не Enterprise. Любая фича взвешивается «нужна для pet?», не «есть в УТ11?».

## 1. Что уже есть (текущая поверхность DSL)

Источник: `internal/dsl/token/token.go`, `internal/dsl/interpreter/builtins.go`, `internal/dsl/interpreter/interpreter.go:328` (evalNew), `internal/dsl/interpreter/collections.go`.

### Синтаксис

- `Процедура / КонецПроцедуры`, `Функция / КонецФункции`
- `Если / Тогда / Иначе / КонецЕсли`
- `Для ... По ... Цикл / КонецЦикла`, `Для Каждого ... Из ... Цикл / КонецЦикла`
- `Возврат`, `Прервать`, `Продолжить`
- `Попытка / Исключение / КонецПопытки`, `ВызватьИсключение`
- Операторы: `+ - * /`, `= <> < > <= >=`, `И ИЛИ НЕ`
- Литералы: `Истина / Ложь`, числа, строки, индексация `[]`
- Дуальный регистр: русские+английские ключевые слова

### Объекты через `Новый` (factory pattern)

- `Массив` / `Array` — Add/Count/Index — `collections.go:10`
- `Структура` / `Structure` — Insert/Get — `collections.go:70`
- `Соответствие` / `Map` — Insert/Get/Delete — `collections.go:146`
- `Запрос` / `Query` — `query_proxy.go`
- `ТабличныйДокумент` / `SpreadsheetDocument` — печатные формы DSL
- `Диаграмма` / `Chart` — ECharts (Plan 06)
- `HTTPСоединение` / `HTTPRequest` — HTTP-клиент (Plan 19)
- `ПисьмоEmail` — отправка почты (Plan 20)

### Builtins (~30 функций)

Строки: `ВРег/Upper`, `НРег/Lower`, `СокрЛП/TrimAll`, `Лев/Left`, `Прав/Right`, `Сред/Mid`, `СтрДлина/StrLen`, `СтрНайти/StrFind`, `Строка/String`, `Число/Number`.

Числа: `Окр/Round`, `Абс/Abs`, `Цел/Int`, `Макс/Max`, `Мин/Min`.

Даты: `Год/Year`, `Месяц/Month`, `День/Day`, `ТекущаяДата/Today`, `ТекущаяДатаВремя/Now`.

JSON: `ПрочитатьJSON/ReadJSON`, `ЗаписатьJSON/WriteJSON`.

Прочее: `Сообщить/Message`, `ВызватьИсключение`.

### Runtime-возможности

- ОбработкаПроведения (через `.posting.os`)
- ПриЗаписи
- Транзакции (Plan 15)
- Inline query language (Plan 14)
- Регламентные задания (Plan 17)
- Отладчик с брейкпоинтами и step-by-step (Plan 99)

## 2. Что я считаю **нужно реализовать** (приоритет 1)

Это минимум, без которого писать pet-конфигурации неудобно или невозможно.

### 2.1. Синтаксис

#### S1. `ИначеЕсли` / `ElseIf`

Сейчас приходится писать вложенные `Иначе Если Тогда ... КонецЕсли КонецЕсли КонецЕсли`. Тривиальное расширение парсера, постоянно нужно.

**Файлы:** `lexer.go` (keyword), `parser.go` (parseIf), `ast.go` (IfStmt с цепочкой).
**Объём:** 1 день.
**Дублирует:** Plan 34 F1.

#### S2. Тернарный оператор `?(условие, истина, ложь)`

1С-синтаксис, экономит 4-5 строк на простом case. Без него короткие подстановки превращаются в полноценные `Если ... Тогда ... КонецЕсли`.

**Файлы:** `parser.go` (parsePrimary), `interpreter.go` (evalExpr).
**Объём:** 0.5 дня.

#### S3. Операторы присваивания `+=`, `-=`, `*=`, `/=`

`Сумма += Стр.Сумма` — каноничный паттерн при обходе табличной части. Сейчас `Сумма = Сумма + Стр.Сумма` — больше шума.

**Файлы:** `lexer.go`, `parser.go`, `interpreter.go`.
**Объём:** 0.5 дня.

### 2.2. Builtins — даты

Без них отчёты и регламентные задания пишутся как костыли с ручной арифметикой.

#### B1. Дата-арифметика

- `НачалоМесяца(Дата)` / `КонецМесяца(Дата)`
- `НачалоГода(Дата)` / `КонецГода(Дата)`
- `НачалоНедели(Дата)` / `КонецНедели(Дата)`  
- `НачалоДня(Дата)` / `КонецДня(Дата)`
- `ДобавитьМесяц(Дата, N)`
- `РазностьДат(Дата1, Дата2, Единица)` где Единица: `"Секунда" | "Минута" | "Час" | "День" | "Месяц" | "Год"`

**Файлы:** `builtins.go`.
**Объём:** 1 день (включая тесты).
**Дублирует:** Plan 34 F6 (частично).

#### B2. Час / Минута / Секунда / ДеньНедели

К дополнению к `Год / Месяц / День`. Тривиально, нужно для регламентных заданий.

**Объём:** 0.5 дня.

### 2.3. Builtins — строки

Сейчас нет даже `Заменить` — это вынуждает писать сами через `Найти+Сред`.

#### B3. Расширенные строковые операции

- `СтрЗаменить(Строка, Что, Чем)` — replace all
- `СтрРазделить(Строка, Разделитель)` → Массив строк
- `СтрСоединить(Массив, Разделитель)` → строка
- `СтрНачинаетсяС(Строка, Префикс)` / `СтрЗаканчиваетсяНа(...)`
- `СтрСодержит(Строка, Подстрока)` (короче чем `СтрНайти > 0`)
- `СтрШаблон(Шаблон, П1, П2, ...)` — `"Привет, %1!"` → подстановка

**Файлы:** `builtins.go`.
**Объём:** 1 день.

### 2.4. Builtins — общего назначения

#### B4. `Пустая(Значение)` / `ЗначениеЗаполнено(Значение)`

Сейчас reference-поле сравнивается через `<> ""` — некрасиво и хрупко. Эти builtins покрывают и nil UUID, и пустую строку, и null.

**Объём:** 0.5 дня.
**Дублирует:** Plan 34 F4.

#### B5. `Формат(Значение, ФормСтрока)`

Без него суммы в шаблонах документов выводятся как `1234.56` вместо `1 234,56 ₽`. Минимальный набор форматов:
- `"ЧДЦ=2"` — две дробные
- `"ЧРГ=' '"` — разделитель тысяч
- `"ДФ='dd.MM.yyyy'"` — формат даты
- `"Л=ru"` — locale

**Файлы:** `builtins.go` (новая `format.go` поверх).
**Объём:** 1–1.5 дня (формат-строки нужно парсить).

#### B6. `ТипЗнч(Значение)` / `Тип(ИмяТипа)`

Нужно для обобщённого кода: «если параметр — Массив, обработать в цикле; если Строка — однократно».

`ТипЗнч(Х) = Тип("Число")` — паттерн 1С.

**Объём:** 0.5 дня.

### 2.5. Runtime

#### R1. Общие модули (`src/common/*.os`)

Без них нет переиспользования кода между сущностями. Каждая `.posting.os` пишет «пересчитать валюту» заново.

**Файлы:** `internal/project/loader.go`, `internal/runtime/registry.go`, `interpreter.go`.
**Объём:** 2 дня.
**Дублирует:** Plan 34 F2.

#### R2. `this.X.Y` — resolve reference fields

`this.Кошелек.Валюта` — без этого нельзя ничего считать в кросс-объектном контексте. Сейчас приходится делать ручной `Запрос`.

**Файлы:** `interpreter.go` (evalSelector + lazy fetch), `runtime/`.
**Объём:** 2 дня. **Гочка:** N+1 запросы, нужен batch-prefetch когда reference обращается в цикле.
**Дублирует:** Plan 34 F3.

#### R3. `ПередЗаписью` для документов и справочников

Обработчик, вызываемый перед `INSERT/UPDATE`, с возможностью отказа. Нужен для:
- автоподстановки полей (имя кошелька из валюты);
- валидации (сумма документа = сумма строк);
- денормализации.

**Объём:** 2 дня.
**Дублирует:** Plan 34 F5.

## 3. Что **полезно**, но не блокер (приоритет 2)

Делаем когда реально упрёмся, не превентивно.

### 3.1. Синтаксис

| # | Конструкция | Что даёт |
|---|---|---|
| S4 | `Выбор Когда X = 1 Тогда ... Когда X = 2 Тогда ... Конец` | switch — длинные цепочки если-иначеесли уходят |
| S5 | Многострочные строки 1С-стиль (`"a"+Символы.ПС+"b"` или ` "a" \| "b" `) | SQL-запросы и шаблоны без + конкатенации |
| S6 | Именованные параметры в вызовах: `ф(Имя = "Х", Возраст = 20)` | вместо позиционных |
| S7 | Деструктуризация структуры в локальные переменные | удобно при возврате нескольких значений |

### 3.2. Builtins

| # | Builtin | Что даёт |
|---|---|---|
| B7 | `Символ(Код)` / `КодСимвола(Строка)` | работа с непечатными символами, переносы |

| B11 | `СтрокаШаблон("$1+$2")` или `Шаблон()` со named-args | i18n / UI-сообщения |
| B12 | `Подождать(СекундЫ)` | в регламентных заданиях |
| B13 | `СгенерироватьGUID()` / `НовыйUUID()` | внешние ключи, идемпотентные операции |
| B14 | `Округление(Число, Точность, Режим)` (с режимами round-half-up/down) | финансовые расчёты |

### 3.3. Runtime

| # | Возможность | Что даёт |
|---|---|---|
| R4 | `Документы.<Имя>.Создать()` + `.Записать()` из DSL | автоматическое создание документов (планирование) |

| R9 | Константы с типом `reference:Имя` | базовая валюта, текущий период |

## 4. Что **не делаем** (явно отказ)

Помечено как ❌ в `Plans/35-1c-feature-inventory.md`. Перечисляю с обоснованием, чтобы вопрос не возникал повторно.

| Конструкция | Почему отказ |
|---|---|
| `#Если Сервер ... #КонецЕсли` | OneBase web-only, нет деления клиент/сервер |
| `#Область / #КонецОбласти` | косметика, IDE решает |
| `ТаблицаЗначений` как полноценный объект (Свернуть/Итог/Выгрузить) | для pet достаточно `Массив` + `Структура`. УТ11 — не наш target |
| `ФиксированнаяСтруктура` / `ФиксированныйМассив` | immutability не критична для pet |
| `Выполнить("код")` (dynamic eval) | безопасность + сложность отладки |
| `ОписаниеТипов()` (динамика типов в runtime) | без СКД нет реального use-case |
| `ЗначениеВСтрокуВнутр / ИзСтроки` (внутренний binary) | используем JSON |
| Web-сервисы (SOAP) / XDTO-пакеты | не для pet |
| HTTP-сервисы внутри конфигурации | у OneBase отдельный REST API (Plan 26) |
| Внешние компоненты (Native C++) | избыточно |
| Многомерные регистры на уровне DSL | через регистр сведений + поля |

## 5. Моё мнение по приоритетам

**Топ-3 что я бы сделал немедленно**, ещё до Plan A (familyBg):

1. **`ИначеЕсли` (S1)** — настолько частая конструкция, что её отсутствие переписывает почти все примеры в TODO.
2. **Расширенные строки (B3)** — `СтрЗаменить` / `СтрРазделить` / `СтрСоединить` нужны постоянно, отсутствие выглядит как «незаконченная платформа».
3. **Даты (B1)** — `НачалоМесяца / ДобавитьМесяц / РазностьДат` — без этого регламентные задания и отчёты по периодам пишутся костылями.

Это 2–3 дня работы и **сразу даёт совсем другую языковую поверхность** при том же количестве объектов.

**Следующая волна** — runtime-фичи (R1 общие модули, R2 двойная точка, R3 ПередЗаписью). Они требуют больше, но без них «сложные» конфигурации действительно невозможны. Это то, что в [[34-dsl-real-world]].

**Не торопиться** с `Формат()` (B5) — выглядит обязательным, но текущие шаблоны печатных форм могут форматировать сами. Полезно для DSL-side, но не блокер.

**Категорически не делать** «всё что в УТ11» — там 2795 общих модулей и `ТаблицаЗначений` в 80% мест. Это **их** архитектура, не наша.
