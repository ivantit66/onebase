### 43.2 — Полный graceful shutdown (приоритет 2, многопольз. критично)

✅ Закрыто 2026-06-25.

- HTTP-часть корректно вызывает `srv.Shutdown(ctx)` в `cli/run.go`, `cli/dev.go`,
  `api/server.go`.
- `internal/scheduler` дренирует активные задания при остановке: ждёт завершения,
  а при истечении shutdown-deadline отменяет job-контексты и фиксирует
  `interrupted` в `_scheduled_runs`.
- `run`/`dev` ждут завершения scheduler-горутины перед выходом из команды, чтобы
  `db.Close()` не сработал раньше финальной записи статуса задания.
- PostgreSQL/SQLite автобэкапы пишутся атомарно: temp-файл + rename, без
  частичного финального файла при прерывании.

### 43.3 — Единый стандарт логирования (приоритет 3)

`log/slog` уже используется в scheduler и частично в runtime, но рядом остаются
точечные `fmt.Println`/`log.Printf`.

Целесообразный остаток:
- Завести единый логгер на `log/slog` с уровнями Error/Warn/Info/Debug.
- Контекстные поля: `request_id`, `entity`, `operation`.
- Заменить прямые `fmt.Println`/`log.Printf` в production-коде.

Не смешивать с 43.2: это отдельная техническая миграция на 2–3 дня, её лучше
делать отдельным PR/срезом, чтобы не размыть shutdown-изменения.

### 43.4 — Раскол монолитов (приоритет 4, по мере необходимости)

✅ Основной долг из исходной формулировки уже снят: `configurator.go`/
`configurator_tmpl.go` больше не являются единственными огромными точками
изменений, UI handlers разнесены по доменным файлам.

Целесообразный остаток:
- не делать механический раскол ради самого раскола;
- при работе рядом дробить новые текущие крупные файлы (`templates.go`,
  `templates_managed.go`, `configurator_tmpl_tree.go`, `handlers_entity.go`)
  только если выделяется самостоятельная доменная граница и есть тестовый якорь.
