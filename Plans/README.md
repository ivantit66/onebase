# Планы развития onebase

Список планов поэтапного развития платформы. Каждый файл — самодостаточный план одной фичи: контекст, YAML/SQL/DSL формат, изменения в коде, тесты, verification, эстимейт.

> ⚠️ **Статусы ниже отстают от факта** — фичи нередко выезжают без обновления этой
> таблицы. Перед тем как считать план «не начатым», сверяйся с кодом:
> `git log --pretty=%s | grep -i "план N"` + наличие файлов/тестов в `internal/`.
> Не доверяй и чек-боксам внутри планов: напр. в `68-report-composition-v2.md`
> отмечено 0/81, хотя этапы A+B уже влиты.
> Сверка от 2026-06-18: планы **59** (компоновка отчётов), **66** (страницы),
> **67** (`ВыполнитьКоманду`) — реализованы; **68** (СКД v2) — этапы A+B готовы,
> остался этап C. Ранее (2026-06-16): **44** (субконто), **39** (виджеты подсистем),
> **57** (ИИ-конфигуратор, этапы 0–3) — тоже реализованы, хотя были помечены ⬜.

## Шаблон нового плана

Каждый план должен содержать разделы:

1. **# Этап N — Название**
2. **## Контекст** — почему делаем, какую проблему решаем
3. **## Синтаксис / YAML** — как фича выглядит для пользователя
4. **## Хранилище / SQL** — изменения в схеме БД (если применимо)
5. **## Изменения в коде** — конкретные файлы и сигнатуры
6. **## Тесты** — какие тесты добавляются (юнит + integration)
7. **## Verification** — как проверить вручную end-to-end
8. **## Эстимейт** — оценка трудозатрат с разбивкой
