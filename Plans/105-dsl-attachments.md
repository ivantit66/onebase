# 105 — Вложения из DSL: ПрисоединитьФайл / СписокВложений / ПутьКВложению / УдалитьВложение

## Проблема

Хранилище вложений (`internal/storage/attachments.go`) было доступно только из
UI и REST v2 — DSL-код (обработки, регламентные задания) не мог ни прикрепить
файл к записи, ни получить путь к уже прикреплённому. Это блокировало сценарии
«сканер папки релизов регистрирует файл вложением карточки релиза» и «отправить
файл вложения письмом/через внешнюю команду».

## Решение

`internal/ui/dsl_attachments.go` — четыре builtin'а, регистрируются в
`buildDSLVars` (доступны обработкам, procrun, заданиям через план 107):

- `ПрисоединитьФайл(Ссылка, ПутьКФайлу[, ИмяФайла]) → ИдВложения` — файл с
  диска (уважает файловую песочницу через новый экспорт
  `interpreter.ResolveSafePath`), имя нормализуется `SanitizeAttachmentName`,
  тип по расширению, лимит размера и allowed_types — как у UI-загрузки;
- `СписокВложений(Ссылка) → Массив Структур {ИД, ИмяФайла, Размер, ТипMIME, Загружен}`;
- `ПутьКВложению(ИдВложения) → строка` — абсолютный путь к файлу хранилища
  (для передачи в ПисьмоEmail.ПрисоединитьФайл / ВыполнитьКоманду);
- `УдалитьВложение(ИдВложения)`.

RLS: запись — «write» владельца, чтение — «read» (checkDSLRowAccess).

## Тесты

`internal/ui/dsl_attachments_test.go` — полный цикл (прикрепить → список →
путь+содержимое → удалить) и регистрация в buildDSLVars.
