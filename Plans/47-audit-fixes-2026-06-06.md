## План реализации (по приоритету)

### Этап 1 — Безопасность (дёшево, высокий эффект)
- [x] 1.1 Path traversal: хелпер `safeBackupPath(dir, file)` — отвергает `..`, разделители, NUL, проверяет `filepath.Rel`. Применён в download/delete/restore/upload. Тест `safe_backup_path_test.go`.
- [x] 1.2 Zip-slip: guard через `filepath.Rel` в цикле распаковки `universal.go`.
- [x] 1.3 `text/template` → `html/template` в конфигураторе. **Реализовано 2026-06-19** (ветка `feature/55-configurator-htmltemplate`, Фаза 1 плана 55): переключение движка + funcmap `js`→`template.JS`; контекстное экранирование закрыло XSS. Init-паника html/template не сработала, churn в тестах нулевой (кириллица не экранируется). Тесты `configurator_xss_test.go` (экранирование в HTML+JS+`{{js}}`), `configurator_render_test.go` (smoke-рендер всех вкладок на rich-данных). Вынос фронта в `go:embed` — Фаза 2 плана 55.

### Этап 2 — Целостность данных
- [x] 2.1 Централизованный `Server.clearMovements` (accumulation + **info** + **account**) вместо разрозненных циклов только по accumulation. Применён в unpost и всех 4 путях удаления (`deleteRecord`, `deleteMarkedAll`, `deleteMarked`, DSL `DeleteRef`). Закрыл orphan-движения info/account-регистров.
- [x] 2.2 panic recovery в `WithTx` (`defer recover()` + rollback + re-panic) для обоих движков.

### Этап 3 — DSL strict mode
- [x] 3.1 Деление на ноль → исключение `userError{"Деление на ноль"}` с номером строки. Тесты `divzero_test.go`. Не-числовые операнды сохраняют прежнее поведение.
- [x] 3.2 Лимит итераций `maxWhileIter` для `NumericForStmt`. `ForEach` оставлен без лимита намеренно: итерируется по уже материализованным коллекциям (ограничены памятью), DoS-вектор — именно `Для i=1 По N`.
- [ ] 3.3 Strict для неизвестных идентификаторов. **Отложено**: рискует сломать существующий прикладной код; нужен флаг strict и аудит конфигураций.

### Этап 4 — БД (отдельно, требует миграций) — **отложено**
- [ ] 4.1 Индексы на `deletion_mark`, ref-полях, датах — требует генерации в `ddl.go` + миграции существующих баз. Отдельная задача.
- [ ] 4.2 `AccountBalances`: `float64` → `decimal` — затрагивает отчётность, нужен отдельный проход с проверкой форматирования.
- [ ] 4.3 `CheckRefs`: консолидация N+1 — низкий приоритет.

### Чистка
- [ ] 5.1 Удалить мёртвый `FormInterpreter`/`parseFormProcedure` — безопасно, но отдельным коммитом.

**Примечание по 1.3:** переключение на `html/template` в 5110-строчном шаблоне с inline-JS рискованно (html/template экранирует иначе внутри `<script>`). Делать осторожно, отдельным проходом с проверкой рендера.
