# Этап 23 — Поиск и быстрые фильтры в списках

**Статус:** ✅ Реализовано

## Контекст

В списках справочников и документов (>100 записей) нет поиска — пользователь вынужден прокручивать. Журналы документов уже имеют фильтры (`internal/storage/journal.go`), но обычные списки — нет. Это базовый UX-фундамент.

## Синтаксис / UX

### Строка поиска (full-text по строковым полям)

```
GET /ui/catalog/Номенклатура?q=стол
→ ILIKE %стол% по полям типа string
```

В интерфейсе: поле «Поиск» над списком (debounce 300ms, без Enter).

### Быстрые фильтры

Боковая панель или строка фильтров над таблицей:

```
GET /ui/documents/Реализация?filter[Покупатель]=<uuid>&filter[Дата_from]=2026-01-01&filter[Дата_to]=2026-03-31
```

Типы фильтров по типу поля:
| Тип поля | Виджет |
|---|---|
| `string` | Текстовое поле (ILIKE) |
| `number` | Диапазон «от / до» |
| `date` | Диапазон дат (date picker) |
| `bool` | Чекбокс / все |
| `reference:X` | Выпадающий список из справочника X |
| `enum:X` | Выпадающий список значений перечисления |

### Пагинация

```
GET /ui/catalog/Номенклатура?page=2&limit=50
```

По умолчанию: `limit=100`. Внизу списка — «Показать ещё» (infinite scroll) или постраничная навигация.

## Изменения в коде

**`internal/storage/crud.go`:**
- добавить параметры `search string`, `filters map[string]any`, `limit, offset int` в `List`
- построение WHERE-условий: `ILIKE` для строк, диапазоны для чисел/дат, `=` для ссылок/перечислений

**`internal/ui/handlers.go`:**
- разобрать query-параметры `q`, `filter[*]`, `page`, `limit` из запроса
- передать в `store.List`

**`internal/ui/templates.go`:**
- компонент строки поиска
- компонент панели фильтров (рендерится по метаданным полей)
- пагинация внизу списка

## Настройка в YAML (опционально)

```yaml
name: Номенклатура
search_fields: [Наименование, Артикул]   # по каким полям искать; по умолчанию — все string
default_sort: Наименование
default_limit: 50
```

## Тесты

- Юнит: `buildWhereClause` с разными типами фильтров
- Интеграционный: список с search=... возвращает только совпадающие записи
