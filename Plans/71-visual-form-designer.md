# План 71 — Визуальный конструктор управляемых форм (issue #164)

WYSIWYG-конструктор управляемых форм: палитра реквизитов → **drag на canvas** →
панель свойств → двусторонняя синхронизация с YAML. По аналогии с
1С:Конфигуратором. Развивает палитру реквизитов #134 и read-only предпросмотр.

## Контекст (отправная точка)
- Редактор формы сейчас: Monaco (YAML) + **read-only** предпросмотр (упрощённый
  серверный рендер в `internal/launcher/forms_tmpl.go`, НЕ рантайм-рендер).
- Палитра #134 вставляет реквизит **текстом** в YAML по эвристике отступов —
  ломко (кривой отступ → невалидный YAML).
- Модель `metadata.FormModule`/`FormElement` богатая (`Walk/FindByID/GenerateID/
  IsContainer`), но есть только разбор **YAML→модель** (`internal/dsl/loader/
  managed_form_loader.go`). Обратной **модель→YAML** с сохранением порядка и
  комментариев НЕТ — это центральная проблема.
- Стек конфигуратора: ванильный JS в Go-шаблонах, без сборки; vendored Monaco/
  SlickGrid/ECharts/Quill по `/vendor/*`.
- Хранение: `forms/<сущность>/<форма>.form.yaml` (+ `.form.os`), ФС или БД.

## Решения (брейншторм)
- **Объём MVP (A):** round-trip + drop нового реквизита из палитры на canvas +
  панель свойств + drop-зоны в группах/страницах. **Откладываем:** перестановку
  существующих элементов (UI), операции с группами/страницами (добавить/удалить/
  переименовать), редактор табличных частей (SlickGrid).
- **Round-trip (A):** канонический документ — дерево `yaml.Node` на сервере.
  Правки хирургические (вставить/изменить узел). Комментарии и порядок ключей
  сохраняются. `FormModule` выводится из узла только для отрисовки.
- **Раскладка (B):** вкладки «Визуально | YAML | Модуль». «Визуально» = палитра +
  холст + свойства во всю ширину. Синхронизация YAML↔холст при переключении и
  вживую.
- **Сосуществование:** YAML-редактор и палитра-вставка остаются; вставка из
  палитры становится **структурной** (через модель/узел), чем чинится хрупкость
  #134 — ничего не теряем.

## Архитектура — серверо-центричная
Клиент не держит своей YAML-логики (нет JS-сборки, yaml.v3 — в Go).
- Правка на холсте → **команда** `{op:"insert"/"setProp"/"move", target, value}` →
  POST `/forms/edit-op` → сервер хирургически меняет `yaml.Node` → пере-сериализует
  YAML + пере-рендерит холст → `{yaml, canvasHtml, selectedId, ошибки}` → клиент
  меняет холст + обновляет Monaco.
- Правка текста YAML на вкладке → перепарс в `yaml.Node` (+валидация) → пере-рендер
  холста. При правке текста источник — текст.
- «Сохранить» → сериализация `yaml.Node` в `forms/.../*.form.yaml` (как сейчас).

## Идентичность элементов
Каждый элемент холста несёт `data-node-id` = путь в дереве узлов
(`elements.2.children.0`). После каждой правки холст пере-рендерится — пути
консистентны с текущим деревом.

## Компоненты
- **Сервер:**
  - `internal/formdoc` — обёртка над `yaml.Node`: `Load/Bytes` (round-trip),
    операции `SetProp/InsertElement/Move`, маппинг узел⇄`FormModule`, адресация по
    `data-node-id`.
  - Интерактивный рендер холста — расширяет текущий `renderManagedFormPreview`
    (`data-node-id`, drop-зоны, выделение).
  - Эндпоинт `/forms/edit-op` + схема редактируемых свойств по виду элемента.
- **Клиент (ванильный JS):** холст (drag из палитры, клик-выбор, drop-зоны),
  панель свойств, синхронизация с Monaco; вкладки «Визуально | YAML | Модуль».

## Слайсы реализации (итеративно, фундамент вперёд)
1. **`formdoc` round-trip** — `Load/Bytes` сохраняют комментарии и порядок
   (golden-тесты). ← фундамент, наибольший риск.
2. **Маппинг узел⇄`FormModule`** + адресация по `data-node-id`.
3. **Операции** `SetProp`, `InsertElement` (структурная вставка), `Move` —
   хирургические мутации узла, комментарии целы.
4. **Интерактивный рендер холста** (`data-node-id`, drop-зоны, выделение).
5. **Эндпоинт `/forms/edit-op`** + обработка ошибок (невалидный YAML, устаревший
   узел).
6. **Клиент:** вкладка «Визуально», панель свойств, drag из палитры, синхронизация
   с Monaco.

## Обработка ошибок
- Невалидный YAML на вкладке YAML → баннер, визуальные правки заблокированы.
- Команда в устаревший узел (документ изменился) → конфликт, клиент перезапрашивает.

## Тестирование (TDD)
- Go: round-trip `formdoc` (комментарии/порядок — golden), каждая операция (узел
  меняется ожидаемо, комментарии целы), маппинг узел⇄`FormModule`; рендер холста
  (по `FormModule` → ожидаемые `data-node-id`/drop-зоны); интеграция (команда →
  YAML).

## Не входит (follow-up)
Перестановка элементов мышью, операции с группами/страницами, редактор ТЧ
(SlickGrid), внешние обработки/формы (отдельный путь).
