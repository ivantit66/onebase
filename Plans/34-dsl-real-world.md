## Что выбрали (по убыванию приоритета)

### F1. `ИначеЕсли` / `ElseIf`

**Источник:** Ч.3 D1. Используется в `списаниеденег.os`, `списаниеденегплан.os` (Plan A).
**Что:** токен `ИначеЕсли` в lexer + AST-узел в parser + обработка в interpreter. Сейчас вынуждены делать вложенные `Иначе Если`.
**Файлы:** `internal/dsl/lexer/lexer.go`, `internal/dsl/parser/parser.go`, `internal/dsl/ast/`, `internal/dsl/interpreter/`.
**Объём:** 1 день. Тривиально, но нужно покрыть тестами 3 кейса.

### F2. Общие модули (`src/common/*.os`)

**Источник:** Ч.3 P8, Ч.4 A3. УТ11 — 2795 модулей. Без общих модулей нет переиспользования кода.
**Что:** загрузчик читает `src/common/*.os` и регистрирует экспортные процедуры/функции в global namespace. Вызов из любого .os-файла: `ОбщиеФункции.ПересчитатьВалюту(...)`.
**Файлы:** `internal/project/loader.go`, `internal/runtime/registry.go`, `internal/dsl/interpreter/`.
**Объём:** 2 дня.

### F3. `this.X.Y` — resolve ссылочных полей (двойная точка)

**Источник:** Ч.3 D9, P4. Используется во многих кейсах: `this.Кошелек.Валюта`, `Стр.Номенклатура.БазоваяЕдиница` и т.п.
**Что:** при чтении `this.X` где X — reference-поле, runtime подгружает связанный объект и даёт доступ к его полям. Кеш в рамках одного сеанса проведения.
**Файлы:** `internal/dsl/interpreter/env.go`, `internal/runtime/`.
**Объём:** 2 дня. Сложность — N+1 запросы, нужен batch-prefetch.

### F4. `Пустая()` для reference-полей

**Источник:** Ч.3 D4, P14. Используется в `поступлениеденег.os`, `списаниеденег.os` (Plan A).
**Что:** метод/builtin `Пустая(ссылка)` или `ссылка.Пустая()` → true для nil UUID. Заменяет неуклюжее `<> ""`.
**Файлы:** `internal/dsl/interpreter/builtins.go`.
**Объём:** 0.5 дня.

### F5. `ПередЗаписью` для документов и справочников

**Источник:** Ч.3 P6, P7. Нужно для автоподстановки полей: имя кошелька из валюты, сумма из табличной части.
**Что:** runtime-событие, вызывается перед `INSERT/UPDATE`. Процедура `ПередЗаписью(Отказ)` с параметром-флагом отказа.
**Файлы:** `internal/runtime/` (uksprite ОбработкаПроведения), `internal/dsl/interpreter/form_interpreter.go`.
**Объём:** 2 дня.

### F6. Функции дат: `НачалоМесяца`, `КонецМесяца`, `ДобавитьМесяц`, `РазностьДат`

**Источник:** Ч.3 P13, Ч.4 Q13. Без них планирование (копирование плана на следующий месяц) не работает.
**Что:** builtins в interpreter:
- `НачалоМесяца(Дата) → Дата`
- `КонецМесяца(Дата) → Дата`
- `ДобавитьМесяц(Дата, Кол) → Дата`
- `РазностьДат(Дата1, Дата2, Единица) → Число` где Единица: "День" / "Месяц" / "Год"

**Файлы:** `internal/dsl/interpreter/builtins.go`.
**Объём:** 0.5 дня.

## Что НЕ берём (отложено)

- **`ТаблицаЗначений`** (Ч.4 L6/L8) — для pet можно обойтись массивами + Struct. УТ11 это не наш target.
- **Создание документов из DSL** (`Документы.Имя.Создать()`) (Ч.3 P2) — большая фича, отдельный план.
- **Подписки на события** (Ч.4 A2) — глобальные обработчики. Сложно, на потом.
- **`ОбработкаЗаполнения` / ввод на основании** (Ч.3 D2, P3) — требует event-системы в runtime. Отдельный план если станет блокером.
- **`Формат()`** (Ч.3 P5, Ч.4 L13) — приятно, но не блокер. Если очень нужно — добавить как один builtin.
- **`ТипЗнч()` / `Тип()`** (Ч.4 L14) — нужно скорее для общих модулей УТ11. Pet обходится без них.
- **Пересчёт валют** (Ч.3 P1) — это бизнес-логика, не платформа. Делается на стороне common module (после F2).
- **Условная компиляция `#Если Сервер`** (Ч.4 L1) — onebase без клиента, не нужно.
