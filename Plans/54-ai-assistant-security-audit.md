## Контекст

ИИ-чат (план 51, F4) даёт модели read-only инструменты данных через tool-use. Доступ
открывается админам и пользователям с флагом `AIDataAccess` (`ui/ai_tools.go:25-31`).
Инструмент `выполнить_запрос` (`ai_tools.go:188`) компилирует и исполняет **произвольный**
read-запрос **без проверки объектного RBAC** (`s.can(...)` не вызывается, в отличие от
обычных list/form-обработчиков). Это задокументировано в коде, но по факту флаг =
«читать всё в базе».

Две грани риска:
1. **Обход RBAC.** Пользователь с `AIDataAccess`, но без права читать «Договоры», получит
   их через ИИ-запрос.
2. **Экфильтрация.** Результаты уходят во внешний LLM-провайдер (Anthropic/z.ai/Gemini).
   Выдача флага = разрешение выгружать любые данные базы третьей стороне в диалоге.

Плюс **нет лимита частоты/стоимости** на `/ui/ai/chat` (`ai_chat.go:30`): один запрос —
до 12 раундов tool-use × вызовы LLM (`tools.go:35`). Вектор cost-DoS, особенно в публичной демо.

> Read-only гарантируется конструкцией языка запросов (нет `INSERT/UPDATE/DELETE` в
> `kwMap`, `query.go:299`) — **это менять не нужно**, оно корректно. Здесь — про
> авторизацию, аудит и стоимость, а не про инъекции.

## Этап 1 — Объектный RBAC для ИИ-запросов (опционально, по настройке)

Сделать так, чтобы `aiRunQuery` уважал права чтения текущего пользователя — но не сломать
сценарий «доверенный аналитик видит всё».

**Дизайн.** Режим доступа ИИ к данным — настройка базы (`_settings`, ключ `ai.data_scope`):
- `admin_only` (дефолт) — как сейчас, но только админ;
- `rbac` — инструменты доступны пользователям, но источники запроса фильтруются по
  `s.can(read)`: до выполнения проверяем, что все таблицы-источники запроса разрешены
  пользователю; иначе возвращаем модели ошибку «нет доступа к объекту X» (она сама
  переформулирует ответ);
- `all` — текущее поведение флага (явный «доступ ко всему»), с предупреждением в UI.

**Как фильтровать источники.** `query.Compile` уже разбирает имена источников
(`preScanMainTable`, `refDims`, типы `РегистрНакопления.X` и т.п.). Вынести из компилятора
список затронутых объектов (`Result.Sources []SourceRef{Kind, Name}`), затем в `aiRunQuery`
до `QueryAll` проверить `s.can(r, kind, name, "read")` по каждому.

| Файл | Изменение |
|---|---|
| `internal/query/query.go` | `Result.Sources` — список (Kind, Name) источников запроса |
| `internal/ui/ai_tools.go` | `aiDataAllowed` учитывает `ai.data_scope`; `aiRunQuery` проверяет `s.can` по `res.Sources` в режиме `rbac` |
| `internal/storage/settings.go` | get/set `ai.data_scope` |
| карточка пользователя (`admin*.go`) | у флага `AIDataAccess` — предупреждение «чтение всех данных + передача во внешний ИИ» |
